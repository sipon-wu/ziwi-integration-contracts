# 知微AI教学助手 — QA 从严回测用例集 V1

> 方法：**功能冒烟 → 数据链路 → 页面一致性 → 边界与安全**
> 原则：测试以**用户视角**发现问题，不以"不崩溃"为通过标准
> 级别：🔴 P0(阻塞) 🟡 P1(严重) 🟢 P2(一般)

## 一、提测前置检查（提测方自检，不过不退QA）

QA 收到提测后，先扫这 3 条，任一不通过则**退回开发侧，不计入测试工时**。

### GATE-01：数据来源真实
| # | 检查项 | 操作 | 通过标准 |
|---|--------|------|---------|
| G01 | 页面无硬编码 mock 数据 | 逐页审查，所有列表/详情数据来自 API 调用 | 没有 `MOCK_*` 、`INIT` 等可识别的假数据变量；API 返回空数据应显示空态，不显示假数据 |
| G02 | 数据跨页一致性 | 设置页改班级→刷新→班级切换页看同一班级 | 数据一致，不出现"设置页有数学、切换页无数学" |

### GATE-02：核心交互可操作
| # | 检查项 | 操作 | 通过标准 |
|---|--------|------|---------|
| G03 | 所有按钮有点击响应 | 逐页点每个按钮 | 弹窗/跳转/Toast 出现，不点完无反应 |
| G04 | 下拉/选择器可展开 | 逐页测每个 `<select>`/下拉框 | 能展开选项、能选中、选中后有反馈 |
| G05 | 表单可输入可提交 | 每个输入框输入后提交 | 保存/提交后有反馈（Toast/列表刷新） |

### GATE-03：状态同步
| # | 检查项 | 操作 | 通过标准 |
|---|--------|------|---------|
| G06 | 全局状态在页面间同步 | TeachingContext 切换学科/年级 → nav 到另一个页面 | 头部/侧边栏/筛选器 显示一致 |
| G07 | 刷新后状态不丢失 | 登录后切学科 → F5 刷新 | 刷新后保持切换后的学科 |
| G08 | 顶部班级/学科不是硬编码 | 查看 AppLayout 头部渲染逻辑 | 头部从 TeachingContext 读取，代码中无`<span>语文 · 四年级 (1班)</span>`这类写死文本 |

## 七、执行状态与闭环移交（必读）

用例不是终点——**每条用例的执行结果必须进入缺陷生命周期闭环**，否则"测出问题"毫无意义。

### 7.1 用例执行状态字段

每条用例在"现状"列之外，执行时需标注动态状态（写进 QA 报告，不污染本文件）：

| 状态 | 含义 | 动作 |
|------|------|------|
| Not Run | 未执行 | — |
| Pass | 通过 | 记入通过数 |
| Fail | 失败 | **开缺陷 → 走第二章 Bug Lifecycle → 直到 Closed** |
| Blocked | 被阻塞（如依赖功能未做） | 标 P0 阻塞，阻断发布 |

### 7.2 从"发现 FAIL"到"闭环"的移交链

```
用例执行 Fail
  → 按第六章格式开缺陷（New）
  → 定级 P0/P1/P2 + 指派（Triaged）
  → 开发修复 + commit（Fixing→Fixed，移交物=commit号）
  → QA 复测原用例转 ✅（Verifying）
  → 同步把本文件对应"现状" ❌ 改 ✅（Closed，闭环证据）
```

**闭环判据**：缺陷 `Closed` 的硬条件是——本用例集里对应行的"现状"从 ❌ 变为 ✅。只改代码不更新用例现状，不算闭环。

- 提测→上线 全流程状态机：见 `../流程闭环/状态迁移与移交规范_V1.md` 第一章
- 缺陷生命周期：同上 第二章
- 部署变更（nginx/SSL）互盯闭环：同上 第三章
